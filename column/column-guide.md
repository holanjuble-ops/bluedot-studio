## ① "마케팅이 고민인 분들께" — 첫 3편 기획

**목표 2가지를 동시에 달성하는 3부작**
1. 마케팅의 **큰 그림**을 그려준다 (왜 → 어떻게 → 증거)
2. 다 읽고 나면 *"이 사람들 실력 있네, 맡겨볼까"* 라는 생각이 들게 한다

| 순서 | 역할 | 한 줄 |
|---|---|---|
| 01 | **왜 (인식 전환)** | 전문직 마케팅 지형이 바뀌었다 — 숏폼이 새 관문인 이유 |
| 02 | **어떻게 (방법론·실력)** | 조회수가 아니라 '문의'를 설계하는 사고법 |
| 03 | **증거 (신뢰)** | 한의원 릴스 2편 → 95만 조회수 실제 사례 (이미 작성됨) |

읽는 순서대로 *세상을 보는 눈 → 우리의 방법론 → 진짜 결과*가 쌓여, 마지막엔 신뢰로 착지합니다.

---

### 01. 왜 지금, 전문직에게 숏폼인가
- **카테고리 / 타입:** `시작 가이드` / `인사이트`
- **후킹(deck):** "검색은 이미 마음을 정한 사람만 옵니다. 숏폼은 아직 고민 중인 사람을 데려옵니다."
- **목적:** 큰 그림 + 인식 전환. 블로그·검색광고·플레이스의 한계를 짚고, 숏폼이 '신뢰를 압축해 전달하는' 새 관문이 된 이유를 설명.
- **본문 구성(블록 outline):**
  - `p` 환자·의뢰인은 더 이상 검색부터 하지 않는다 — 피드에서 먼저 만난다
  - `h2` 블로그·검색광고가 예전만큼 안 통하는 이유
  - `list` 광고비 상승 · 광고 불신 · 플레이스 상위노출 경쟁 심화
  - `h2` 숏폼은 '신뢰'를 30초에 압축한다
  - `p` 얼굴·말투·전문성이 한 편에 담긴다 → 처음 보는 사람도 원장님을 '안다'고 느낀다
  - `quote` "정보는 검색으로 얻지만, 신뢰는 얼굴에서 옵니다."
  - `h2` 그래서 무엇부터 해야 하나
  - `p` 다음 글(02편) 예고로 자연스럽게 연결
- **실력 증명 포인트:** 시장 진단의 정확함 → "이 회사, 전문직 시장을 안다"

### 02. 조회수가 아니라 '문의'를 만드는 콘텐츠
- **카테고리 / 타입:** `시작 가이드` / `인사이트`
- **후킹(deck):** "조회수 10만이 문의 0건일 수 있습니다. 숫자가 아니라 구조를 보세요."
- **목적:** 통찰 + 방법론 공개. 블루닷이 *다르게 생각한다*를 보여주는 글. 전환을 설계하는 프레임워크(타겟 → 후킹 → 캡션)를 명료하게 제시.
- **본문 구성:**
  - `p` 조회수는 터졌는데 문의가 없는 계정의 공통점
  - `h2` 1. 누구에게 말하는지부터 좁힌다 (타겟)
  - `h2` 2. 첫 3초에서 멈추게 만든다 (후킹)
  - `h2` 3. 마지막 한 줄이 문의를 만든다 (캡션·CTA)
  - `stat` `[['3초','이탈이 갈리는 시간'], ['1줄','문의를 만드는 캡션'], ['×2.4','문의 전환 차이']]`
  - `quote` "잘 만든 영상이 아니라, 잘 설계된 영상이 문의를 만듭니다."
- **실력 증명 포인트:** 재현 가능한 방법론이 있다 → "감이 아니라 시스템으로 일하는구나"

### 03. 한의원 릴스 2개로 95만 조회수, 그 안에 숨은 후킹 공식  *(이미 작성됨)*
- **카테고리 / 타입:** `한의원` / `성공 사례`
- 현재 `col-detail.jsx`의 `ARTICLE` 내용이 이 글입니다. **그대로 03편으로 사용**하면 됩니다.
- 01·02에서 말한 원칙(후킹·전문성·캡션)이 실제로 통했음을 보여주는 증거편 — 3부작의 착지점.

## ② 남은 2편 제안 (총 5편 완성)

전체 라이브러리를 넓히고, 전문직 특유의 '반론'을 미리 해소하는 두 편을 추천합니다.

### 04. (추천) 의료·법률 광고 심의, 숏폼은 어디까지 가능한가
- **타입:** `제작 가이드` / `인사이트` · **대상:** 병의원 · 법률
- **왜 강력한가:** 전문직은 *규제를 모르는 대행사*를 가장 무서워합니다. 의료광고법·변호사 광고규정을 아는 회사라는 신호 = 즉각적인 안심. 경쟁사 대비 확실한 차별화 포인트.

### 05. (추천) 카메라 울렁증이어도 괜찮습니다 — 대본 없이 촬영하는 법
- **타입:** `제작 가이드` · **대상:** 전 직역 공통
- **왜 강력한가:** 전문직 1순위 반론 *"난 말주변이 없어서…"* 를 정면 해소. 진입장벽을 낮춰 상담 신청으로 연결.

**대안 후보(원하면 교체):**
- 블루닷 프로세스 공개 — 기획·방문촬영·편집·업로드 4단계 (투명성/신뢰)
- 2026 인스타 알고리즘 변화 — 전문직 계정이 알아야 할 것 (트렌드/시의성)
- 업종별 사례 2탄 — 치과 / 법률 등 다른 직역 성과

## ③ 칼럼 한 편을 추가하는 법 — 데이터 구조 총정리

### 파일 지도
| 파일 | 역할 |
|---|---|
| `column/cols-shared.jsx` | `COLS[]` — **목록 페이지** 카드/리스트 메타데이터 (전체 칼럼) |
| `column/col-page.jsx` | `STARTER_PAGE[]` — **"마케팅이 고민인 분들께" 추천 3편** 카드 전용 |
| `column/col-detail.jsx` | `ARTICLE` — **개별 글 본문** (현재 1편만 하드코딩) |
| `column/칼럼.html` | 목록 페이지 진입점 |
| `column/칼럼 상세.html` | 상세 페이지 진입점 |

### ⚠ 가장 먼저 해결할 구조적 한계
지금은 상세 페이지가 **단 1편(`ARTICLE`)만** 보여줍니다. 목록의 모든 링크가 동일한 `칼럼 상세.html`로 가기 때문에, 5편을 각각 띄우려면 **라우팅을 먼저 잡아야** 합니다.

**권장 방식 (가장 깔끔):**
1. 글 본문들을 `id`로 묶은 객체로 분리 — 예: `column/articles.jsx`에 `window.ARTICLES = { 1: {...}, 2: {...}, ... }`
2. `col-detail.jsx`가 URL의 `?id=` 를 읽어 해당 글을 렌더 — `new URLSearchParams(location.search).get('id')`
3. 목록·추천 카드의 링크를 `칼럼 상세.html?id=1` 형태로 변경 (현재 고정 `href="칼럼 상세.html"`)
4. `COLS[]`의 `id`와 `ARTICLES`의 키를 일치시키면 끝

### 글 1편에 필요한 데이터 3종

**(A) 목록 메타 — `cols-shared.jsx`의 `COLS[]`에 한 줄 추가**
```js
{
  id: 7, cat: '병의원', type: '인사이트',
  title: '왜 지금, 전문직에게 숏폼인가',
  excerpt: '한 줄 요약 — 목록 카드에 노출됩니다.',
  date: '2026.06.19', read: 6
}
```

**(B) 추천 3편 카드 — `col-page.jsx`의 `STARTER_PAGE[]` (첫 3편만 해당)**
```js
{ no: '01', tag: '시작 가이드', title: '왜 지금, 전문직에게 숏폼인가',
  desc: '카드에 보이는 짧은 설명.' }
```
> 추천 카드 이미지는 image-slot(`cp-hs-0/1/2`)으로 드래그&드롭해 채웁니다.

**(C) 본문 — `ARTICLE`(또는 권장 구조의 `ARTICLES[id]`) 객체**
```js
{
  cat: '병의원', type: '인사이트', date: '2026.06.19', read: 6, author: '블루닷 에디터',
  title: '왜 지금, 전문직에게 숏폼인가',
  deck: '글 상단 큰 요약 한두 문장.',
  blocks: [ /* 아래 블록 치트시트 참고 */ ]
}
```

### 본문 블록 타입 치트시트 (`blocks[]`)
```js
{ t:'p',     text:'문단. **별표**로 감싸면 노란 형광펜 강조' }
{ t:'h2',    text:'소제목' }
{ t:'quote', text:'인용문 (파란 좌측 바)' }
{ t:'list',  items:['항목1','항목2'] }                       // 파란 점 불릿
{ t:'stat',  items:[['95만+','두 편 합산 조회수'], ['+3,000','늘어난 팔로워']] }  // 숫자 콜아웃
{ t:'reels', images:['../images/a.png','../images/b.png'], caption:'설명' }      // 9:16 2~3컷
{ t:'image', src:'../images/x.png', caption:'설명' }          // 본문 중간 16:9
```

### 이미지 입력 방법 2가지
- **image-slot** (목록·추천·히어로 이미지): 화면에서 직접 드래그&드롭하면 슬롯 `id` 기준으로 저장됩니다. 코드 수정 불필요.
- **`<img src>`** (`reels`/`image` 블록): 실제 파일을 `column/../images/` 에 넣고 경로(`../images/파일명.png`)를 지정해야 합니다.

### 브랜드 보이스 규칙 (작성 시 필수)
- 합쇼체(…합니다). CTA만 부드럽게 해요체 허용
- 숫자 우선 · 천 단위 콤마(1,240,000) · 큰 수는 만/억(124만 뷰)
- **이모지 금지 · 느낌표 금지(헤드라인)**. 허용 기호: `—` `·` `→` `●`
- 영문은 Sentence case. 헤드라인 끝 마침표 없음, 본문은 마침표

## ④ Cowork에 붙여넣을 프롬프트

```
블루닷스튜디오 웹사이트의 칼럼 5편을 완성하려고 해.
현재 구조: 목록은 column/cols-shared.jsx의 COLS[], 추천 3편은
column/col-page.jsx의 STARTER_PAGE[], 본문은 column/col-detail.jsx의
ARTICLE 한 편만 하드코딩돼 있고 목록 링크가 전부 같은 칼럼 상세.html로 가.

먼저 ①상세 라우팅부터 잡아줘:
- 글 본문을 id로 묶은 구조(window.ARTICLES = {1:{...}, ...})로 분리
- col-detail.jsx가 URL ?id= 를 읽어 해당 글을 렌더
- COLS[]·추천카드의 링크를 "칼럼 상세.html?id=N"으로 변경

그다음 아래 5편을 작성해줘 (기획안은 column/칼럼-운영가이드.md 참고):
01 왜 지금, 전문직에게 숏폼인가 (시작 가이드/인사이트)
02 조회수가 아니라 '문의'를 만드는 콘텐츠 (시작 가이드/인사이트)
03 한의원 릴스 2개로 95만 조회수 (한의원/성공 사례 — 기존 ARTICLE 재사용)
04 의료·법률 광고 심의, 숏폼은 어디까지 가능한가 (제작 가이드)
05 카메라 울렁증이어도 괜찮습니다 — 대본 없이 촬영하는 법 (제작 가이드)

블록 타입과 브랜드 보이스 규칙은 column/칼럼-운영가이드.md 와
col-detail.jsx 상단 주석을 그대로 따라줘. 디자인 시스템(_ds/)을 벗어나지 말 것.
이모지·느낌표 금지, 합쇼체, 숫자 우선.
```
